HTML 原型分享配置

同事原型分享配置引导

这个页面用于配置本机发布权限。默认只生成本地 HTML；只有在说“分享原型”时，Codex 才会发布到统一托管仓库。

先看这个限制

这个页面不能自动帮你拿到 GitHub 写权限。仓库 owner 必须先在 GitHub 上手动邀请你成为 collaborator。你接受邀请后，才能发布原型。

Step 1. 接受 GitHub 仓库邀请

  1. 把你的 GitHub 用户名发给管理员。
  2. 管理员在仓库里给你加 Write 权限。
  3. 你在 GitHub 邮件或通知里接受邀请。

Step 2. 登录 GitHub CLI

gh auth login -h github.com
  • Git protocol 选 HTTPS
  • Authenticate Git with your GitHub credentials 选 Y
  • Authentication method 选 Login with a web browser

Step 3. 配置环境变量

把下面这段写入你的终端配置文件。

echo 'export PROTOTYPE_HOSTING_REPO=wanggeng826-bot/seabost-prototype-hosting' >> ~/.zshrc echo 'export PROTOTYPE_BASE_URL=https://wanggeng826-bot.github.io/seabost-prototype-hosting' >> ~/.zshrc source ~/.zshrc

Step 4. 自检

gh auth status echo $PROTOTYPE_HOSTING_REPO echo $PROTOTYPE_BASE_URL npm run prototype:publish -- --help

Step 5. 实际使用

  • 生成原型：只生成本地 HTML 文件
  • 分享原型：发布到 GitHub Pages，并返回在线链接
